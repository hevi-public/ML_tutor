Data & features: rows, columns, and dots

Every dataset is a notebook: one row per thing, one column per fact about it. Learn to see the notebook and the chart as the same object.

Before you start Vectors & matrices helps — a row of the notebook is a vector.

The idea

Back to the ice cream stand. Your notebook has one row per day, with three columns: the temperature, whether it was a weekend, and how much you sold. In ML vocabulary:

  • Each row is a data point — one example.
  • The columns you know in advance (temperature, weekend?) are features — what the model predicts from.
  • The column you want to predict (sales) is the label — the answer key.

That's it. "Dataset" = notebook. "Features" = the known columns. "Label" = the answer column. The whole of supervised learning is: use the filled-in rows to predict the answer column for rows that don't have one yet.

The weekend column is a third direction the flat screen can't draw — with 100 features, a data point is a dot in 100-dimensional space. Nobody can picture that; the trick is that the math (dot products, distances) works the same in 100 dimensions as in 2. You never need to picture it.

Where models actually win or lose

A dull truth with expensive consequences: most real ML work is work on the data, not the model. Things practitioners check before touching any algorithm:

  • Is the label honest? If "sales" was written down from memory a week later, the answer key itself is noisy.
  • Do the features carry the signal? Temperature clearly relates to ice cream sales. The day's lottery numbers don't — no model can squeeze prediction out of features that contain nothing.
  • Is the notebook representative? If every row is from summer, the model learns summer rules and fails in winter, confidently. The data decides what the model can know.
  • Units and scales. A feature measured in millimeters and another in kilometers will fight; putting features on comparable scales (e.g. subtracting the mean, dividing by σ — last page's two numbers!) is routine hygiene.
