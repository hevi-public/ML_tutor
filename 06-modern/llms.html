Unit 6 · Modern ML

Large language models: next-word prediction, taken seriously

Underneath the conversation is a machine doing one thing: given the text so far, score every possible next token. Everything else follows from doing that astonishingly well.

Before you start Attention and embeddings — an LLM is those, stacked and scaled.

First: what's a token?

Models don't quite read words. Text is chopped into tokens — pieces from a fixed menu of a hundred thousand or so, learned by finding the most reusable fragments in a mountain of text. Common words get their own token; rarer words get built from parts (which is why a model can handle "unbreakableness" without ever having a token for it).

This toy menu has ~30 pieces (real ones: ~100,000; same greedy idea). Type "walking talkers" — clean splits. Type "xylophone" — nothing on the menu matches, so it shatters into characters (red). Fun fact with real-world bite: this is why models are weirdly bad at counting letters in a word — they never see letters, only these chunks.

The machine itself

A large language model is a transformer trained on one task: given the tokens so far, predict the next one. Output: a probability for every token on the menu. Pick one (most-likely, or sample with some randomness — the dial called temperature), append it, repeat. Every chatbot reply you've ever read was generated one token at a time this way.

Why does mere next-token prediction produce something that can explain, translate, and write code? Because predicting well forces learning. To continue "the capital of Finland is", you must know it. To continue a half-written function, you must model what the code does. Compress the internet into prediction weights and enormous amounts of world-knowledge, grammar, and reasoning-shaped patterns get dragged in — not because anyone asked for them, but because they reduce the loss (yes — the same log loss you met in logistic regression, summed over trillions of tokens).

Click "most likely" repeatedly: the text gets stuck in a loop (greedy decoding does that — to real LLMs too!). Restart, use the dice: livelier, occasionally silly. That's the temperature trade-off you see in every LLM API. This toy looks back exactly ONE word; an LLM looks back at your whole conversation through attention, with ~10¹¹ knobs instead of a count table. The generation loop, though — score, pick, append, repeat — is identical.

The honest caveats, briefly

  • It predicts plausible, not true. When the model doesn't know, the most probable continuation is often a confident-sounding guess — the failure politely called hallucination. It's not lying; it's doing exactly its job on the wrong occasion.
  • The context window is real memory limits. The model sees a fixed number of recent tokens. Nothing outside it exists — there is no other memory between conversations.
  • Data is destiny, again. Unit 2's lesson at planetary scale: the model inherits whatever the training text contains — brilliance, bias, and boilerplate alike.
