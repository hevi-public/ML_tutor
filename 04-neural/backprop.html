Backpropagation: blame, passed backwards

The algorithm that trains every neural network on Earth is a bookkeeping trick: compute forward, then walk the recipe in reverse, handing each step its share of the blame.

Before you start Slopes & gradients (the nudge question) — backprop is that question, answered wholesale.

The idea

Training needs to know: "if I nudge this weight, what happens to the loss?" — for every one of possibly billions of weights. Measuring each by actually nudging it would take one full network-run per weight. Hopeless.

Backpropagation gets every answer in just two passes. The insight: a network is a recipe of small steps (multiply, add, bend), and each step knows its own local slope. The chain rule — the only calculus fact backprop needs — says nudge-effects along a chain multiply: if a nudge to w moves z by 2×, and a nudge to z moves the loss by 3×, then w moves the loss by 6×. So: run the recipe forward and save every intermediate value; then walk backward from the loss, multiplying local slopes as you go. Arriving at each weight, the accumulated product IS its gradient. Two passes, all answers.

The same trick, at scale

Everything you just stepped through scales without changing character:

  • A billion-parameter network is still a recipe of multiply/add/bend steps — just a very long one. Forward saves values; backward multiplies local slopes. Two passes, every gradient, always.
  • Where paths branch and re-join, blame simply adds along the joining paths. (You saw a hint: z's blame flowed unchanged to both m and b.)
  • Modern frameworks (PyTorch, JAX) are, at their core, industrial-strength versions of this bookkeeping: they record the recipe as you compute, then replay it backwards. It has a lovely name — automatic differentiation.

The code

Show the autodiff engine (yes, the whole thing)

Each Value remembers how it was made and how to pass blame to its parents. backward() visits the recipe in reverse:

// assets/js/ml/autodiff.js — abridged, but faithfully
mul(other) {
  const out = new Value(this.data * other.data, [this, other], "×");
  out._backward = () => {
    this.grad  += other.data * out.grad;  // chain rule: local slope × incoming blame
    other.grad += this.data * out.grad;
  };
  return out;
}

sigmoid() {
  const s = 1 / (1 + Math.exp(-this.data));
  const out = new Value(s, [this], "σ");
  out._backward = () => { this.grad += s * (1 - s) * out.grad; };
  return out;
}

backward() {
  const topo = topologicalOrder(this);  // recipe order
  this.grad = 1;                        // the seed: dL/dL = 1
  for (let i = topo.length - 1; i >= 0; i--) topo[i]._backward();
}

That's the entire mechanism. PyTorch's version is faster and handles tensors, but the soul is identical — in Unit 8 you'll build this file yourself from a blank page.
