Unit 3 · Classical ML

Logistic regression: a line that gives odds

Take linear regression, bend its output into a probability, and you have the workhorse of yes/no prediction — still, decades on, the first model every practitioner tries.

Before you start Linear regression (this page reuses its recipe) and judging classifiers (this model outputs exactly the kind of scores you thresholded there).

The idea

Suppose you're predicting a yes/no: will this customer buy ice cream? The linear recipe — multiply each feature by a weight, add up — still produces a useful number: strongly positive means "very buy-ish", strongly negative "very not". But it ranges over all numbers, and a probability must live between 0 and 1.

So we pass the score through a squashing function: very negative scores squash toward 0, very positive toward 1, and a score of exactly zero lands at 50/50. The squasher is called the sigmoid (it's S-shaped), and the whole assembly is logistic regression. Despite the name, it's a classifier — the "regression" is a historical fossil.

Geometrically, the model draws one straight line through feature-space: the 50/50 line, its decision boundary. Everything on one side leans yes, the other no — with confidence growing the farther you go from the line.

The math (open as much as you like)

The idea in words

Score the input with the linear recipe. Squash the score into 0–1 with the S-curve. Grade the model not on "right or wrong" but on how much probability it gave the truth — confidently wrong costs a lot, mildly wrong costs a little. Then gradient descent, exactly as before.

Show the math

The prediction:

\hat{p} = \sigma(\mathbf{w} \cdot \mathbf{x} + b), \qquad \sigma(z) = \frac{1}{1 + e^{-z}}
In plain English The predicted probability is: the linear score (weights dot inputs, plus offset), squashed by the S-curve σ. The squasher itself: one divided by (one plus e-to-the-minus-score) — big scores → near 1, big negative scores → near 0.

    The grade — log loss:

    L = -\frac{1}{n}\sum_{i=1}^{n}\Big[\, y_i \log \hat{p}_i + (1 - y_i)\log(1 - \hat{p}_i) \,\Big]
    In plain English For each example: if the truth was yes (y=1), the cost is "minus the log of the probability you gave yes"; if no, same for no. Give the truth 99% and pay almost nothing; give it 1% and pay dearly. Average over everyone.

      Why not just mean squared error? Log loss's punishment for confident wrongness is what keeps the model honest about uncertainty — and it happens to make the gradient astonishingly tidy: the nudge for each weight is just (predicted probability − truth) × input. That "error × input" pattern will return, verbatim, inside neural networks.

      The code

      Show the JavaScript (from this site's own mini-library) 
      // assets/js/ml/models.js — the actual code running in the demo above
static sigmoid(z) { return 1 / (1 + Math.exp(-z)); }

predictProba(x) { return ML.LogReg.sigmoid(dot(this.w, x) + this.b); }

trainStep(points, lr = 0.5) {
  const gw = new Array(this.w.length).fill(0);
  let gb = 0;
  for (const p of points) {
    const err = this.predictProba(p.x) - p.y;   // probability − truth
    for (let i = 0; i < gw.length; i++) gw[i] += err * p.x[i];
    gb += err;
  }
  for (let i = 0; i < this.w.length; i++) this.w[i] -= lr * gw[i] / points.length;
  this.b -= lr * gb / points.length;
}
