Linear regression: teaching a line to make predictions

The simplest machine that learns — and the one that quietly contains most of the big ideas in machine learning.

Before you start You only need to be comfortable reading a dot chart (a scatter plot). Any math that appears below is optional, collapsible, and explained symbol by symbol.

The idea

Imagine you run an ice cream stand. You've noticed that hotter days mean more sales, and you've kept notes: each day is one dot on a chart — temperature across, sales up.

Now you'd like a rule of thumb: "for any temperature, roughly how much will I sell?" The simplest possible rule is a straight line drawn through your dots. To predict a day's sales, find the temperature on the line and read off the answer.

That's linear regression. The only question is: which line? Some lines clearly summarize the dots better than others. Machine learning's answer: give each candidate line a grade — how far off its predictions are from the real answers — and find the line with the best grade. That grade is called the loss (lower is better, like golf).

How the computer "learns" the line

When you pressed the button, the computer didn't try every possible line. It did something smarter, and this three-step loop is the heart of most machine learning:

  1. Guess — start with any line at all.
  2. Grade — measure how bad the guess is (the loss).
  3. Adjust — nudge the two sliders in whichever direction makes the grade a little better. Repeat.

The clever part is step 3: the math tells the computer which direction to nudge, without trial and error. This nudging process is called gradient descent — "descent" because it walks downhill toward smaller mistakes. You'll meet it properly in Unit 4; today it's enough to have seen it work.

The math (open as much as you like)

The idea in words

The line's prediction is: take the input, multiply by how much it matters, then add a starting offset. The grade is: for each dot, measure the miss, square it (so misses above and below the line both count, and big misses count a lot), then average over all dots.

Show the math

The prediction rule, written in symbols:

\hat{y} = w x + b
In plain English The prediction (y-hat) is the input times a weight, plus a starting offset.

    And the grade — the mean squared error:

    L = \frac{1}{n} \sum_{i=1}^{n} \left(y_i - \hat{y}_i\right)^2
    In plain English The loss is the average, over all n dots, of (real answer minus prediction) squared.

      Tap any symbol chip (or the symbols in the equations) to hear its name and what it stands for.

      Show the derivation (how "which way to nudge" is computed)

      To know which way to nudge w, ask: "if I increase w a tiny bit, does the loss go up or down?" Calculus answers exactly that question; the answer is called a derivative (∂, said "partial"):

      \frac{\partial L}{\partial w} = -\frac{2}{n} \sum_{i=1}^{n} x_i \left(y_i - \hat{y}_i\right)
      In plain English "How the loss changes as the weight changes" is found by combining every dot's miss, with bigger inputs getting more say. If this number is positive, the loss grows when w grows — so nudge w down. If negative, nudge w up.

      Then update both numbers a small step in the downhill direction, and repeat:

      w \leftarrow w - \eta \, \frac{\partial L}{\partial w} \qquad b \leftarrow b - \eta \, \frac{\partial L}{\partial b}
      In plain English New weight = old weight, minus a small step (eta, the "learning rate" — how big each nudge is) in the uphill direction. Minus uphill = downhill.

        Don't worry if the derivative itself feels like magic — Unit 1 builds it from scratch, gently. The takeaway here: the loss tells the computer not just how wrong it is, but which direction to move to be less wrong.

        The code

        Show the JavaScript (this is genuinely all of it) 
        // points: [{x, y}, ...] — the dots on the chart
let w = 0, b = 0;             // start with any line at all (a flat one)
const eta = 0.008;            // learning rate: how big each nudge is

for (let step = 0; step < 5000; step++) {
  // Grade the current line, and note which way each slider should move
  let gradW = 0, gradB = 0;
  for (const p of points) {
    const miss = p.y - (w * p.x + b);   // real answer minus prediction
    gradW += -2 * p.x * miss;
    gradB += -2 * miss;
  }
  // Nudge both numbers a small step downhill
  w -= eta * (gradW / points.length);
  b -= eta * (gradB / points.length);
}
// w and b now describe the best-fitting line

        Where this goes next

        Everything you just did scales up. Real problems have many inputs instead of one (temperature and day of week and location…) — that's still linear regression, just with one weight per input. And when a straight line is too simple for the pattern, we bend it — which is where neural networks begin (Unit 4).
