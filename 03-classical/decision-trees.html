Decision trees & forests: twenty questions, then a crowd of them

A model you can read like a flowchart — and the trick that turns a mob of unreliable flowcharts into one of the most reliable tools in ML.

Before you start Overfitting — trees are the most visual overfitters in all of ML, and this page leans on that.

The idea

Play twenty questions with the data: "Is the humidity under 4.1? If yes — is the temperature under 2.9? If yes — it's class A." A decision tree is exactly that flowchart, grown automatically: at each step the algorithm tries every possible yes/no question about every feature and keeps the one that best unmixes the classes — the split after which each side is as single-colored as possible. Then it repeats inside each side, until the groups are pure or a depth limit says stop.

Because every question is about one feature at a time ("is x ≤ 4.1?"), a tree's map of the world is all straight fences at right angles — little boxes. And because it can keep asking questions, a deep tree can build a private box around every single dot. You know the name for that.

Crank depth to 10 and find the absurd little boxes built around single stray dots. Then press "plant a forest": 25 deep trees, each trained on a slightly different random re-sample of the dots, voting together. Each tree still memorizes noise — but they memorize different noise, and the vote averages it away. Deep color = near-unanimous; pale = the trees disagree.

Why the mob beats the expert

That button is the biggest idea on this page, worth saying carefully. A single deep tree is a jumpy model — re-deal the data and it draws a different map (high variance, from the overfitting page). The random forest recipe:

  • Train many deep trees, each on a random re-sample of the data (drawn with replacement — so each tree sees a slightly different world).
  • To predict: let every tree vote, majority rules.

Individual jumpiness, being partly random, points in different directions — so it cancels in the average, while the shared signal doesn't. The same trick works far beyond trees; combining many models is called an ensemble, and some flavor of it wins most tabular-data competitions to this day. The price: you can no longer read the model like a flowchart. Interpretability was traded for steadiness.

How "best question" is scored (the one formula on this page)

Each candidate split is judged by how mixed its two groups remain. The standard mixedness score is Gini impurity: for a group, sum each class's share-squared and subtract from 1. A pure group scores 0; a 50/50 group scores 0.5. The tree keeps the question whose two sides have the lowest size-weighted average mixedness — greedily, one question at a time, never looking ahead. (Greedy isn't optimal, but it's fast and works — a very ML trade.)
