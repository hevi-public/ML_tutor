Convolutions: one small question, asked everywhere

How networks see. The idea fits in a sentence: slide a tiny pattern-detector across the whole image and write down where it fires.

Before you start The neuron — a convolution is that same weighted sum, on a leash.

The idea

Feeding an image to the playground's MLP would mean one weight per pixel per neuron — millions of knobs that would have to separately learn that a cat at the left of a photo is the same cat as at the right. Wasteful, and it ignores what images are like: the meaningful stuff is local (edges, corners, textures) and it can appear anywhere.

The fix: make one tiny neuron — say 3×3 pixels' worth of weights, called a kernel (or filter) — and slide it across the whole image, computing its weighted sum at every position. The result is a new image showing where that little pattern occurs. This slide-and-sum is a convolution, and a network built from layers of them is a convolutional neural network (CNN).

Two birds with one stone: the same 9 weights serve every position (a cat is a cat anywhere), and 9 weights instead of millions (the leash).

Compare "vertical lines only" and "horizontal lines only" on the house: same image, same machinery, 9 different numbers — and completely different features found. Now the punchline: in a CNN, nobody designs these kernels. They're weights. Backprop tunes them, and edge-detectors like these reliably emerge in the first layer on their own — because edges are what images are made of.

Stacking the idea

One convolution layer finds simple things (edges, spots of color). The next layer convolves over those results — so its patterns are patterns of edges: corners, curves, textures. Stack again: eyes, wheels, windows. Again: faces, cars, houses. Each layer's vocabulary is built from the previous layer's, which is the "deep" in deep learning doing real work — and it mirrors, loosely, how visual cortex organizes itself.

Two supporting parts you'll meet in every CNN diagram:

  • Many kernels per layer. A layer doesn't learn one detector, it learns dozens (one output image per kernel — each output is called a feature map). "64 channels" = 64 different little questions asked everywhere.
  • Pooling / downsampling: periodically shrink the maps (e.g. keep the strongest response in each 2×2 patch). Detail fades, but "is the pattern here-ish?" survives — which buys tolerance to small shifts and keeps compute sane.
